# 日付抽出ルール: 月のみで年の指定がない場合の処理

## ルールID: `date_month_only_default_year`

## 適用対象
- `period.start_date` フィールド
- `period.end_date` フィールド

## ルール内容
文書に **月のみ**（例: 「7月」「8月」「即日」）が記載され、年が明示されていない場合、
**現在年（システム日付の年）をデフォルトの年として使用する。**

## 具体ロジック

```
入力: 「7月」  (現在日付: 2026年6月)
  → 7月 >= 6月(現在月) → 当年扱い
  → 出力: 2026-07-01

入力: 「5月」  (現在日付: 2026年6月)
  → 5月 < 6月(現在月) → 過ぎた月は翌年扱い
  → 出力: 2027-05-01
```

## 実装箇所

| モジュール | ファイル | 関数/行 |
|---|---|---|
| ルールエンジン | `src/run_pipeline.py` | `extract_period()` / `process_excel()` |
| LLM プロンプト | `src/run_llm_pipeline.py` | SYSTEM_PROMPT 内ルール5 |
| 共通ユーティリティ | `src/common/utils.py` | `convert_jp_date()` |

## 適用例

| 原文 | 現在日付 | 抽出結果 |
|---|---|---|
| 7月〜長期 | 2026-06-28 | start_date: 2026-07-01, long_term: true |
| 2026/08/01〜 | — | start_date: 2026-08-01 |
| 即日参画 | 2026-06-28 | start_date: 2026-06-28 |
| 2025/7/1〜2026/3/31 | — | start_date: 2025-07-01, end_date: 2026-03-31 |
| 8月〜 | 2026-06-28 | start_date: 2026-08-01 |
| 2月〜 | 2026-06-28 | start_date: 2027-02-01 (翌年) |

## 補足

- 「即日」「Immediate」「随時」の場合はシステム日付を開始日とする
- 「長期」フラグは別途 `period.long_term = true` で保持
- 年が明示されている場合は明示された年を優先（上書きしない）
